# 前端响应式开发规范

## 1. 响应式断点标准

项目使用 Ant Design 的 Grid 系统断点标准：

| 断点 | 尺寸 | 设备类型 | 说明 |
|------|------|---------|------|
| xs | < 576px | 手机竖屏 | 单列布局 |
| sm | 576px - 767px | 手机横屏 | 1-2列布局 |
| md | 768px - 991px | 平板竖屏 | 2列布局 |
| lg | 992px - 1199px | 平板横屏/小笔记本 | 3列布局 |
| xl | 1200px - 1599px | 桌面显示器 | 3-4列布局 |
| xxl | ≥ 1600px | 大屏幕显示器 | 4列及以上布局 |

## 2. 布局组件响应式要求

### 2.1 侧边栏 (Sider)

```typescript
// 响应式宽度配置
const siderWidth = screens.xl ? 240 : screens.lg ? 220 : 200;

// 自动折叠策略
- xs: 完全折叠，通过汉堡菜单展开
- sm/md: 可选折叠
- lg+: 默认展开
```

**要求：**
- 移动端 (<768px) 默认折叠，提供折叠/展开按钮
- 移动端侧边栏使用固定定位，覆盖主内容区域
- 侧边栏菜单文字不宜过长，折叠后只显示图标或首字

### 2.2 头部 (Header)

```typescript
// 响应式配置
- padding: xs ? '0 16px' : '0 24px'
- 标题字体: xs ? 16px : 18px
- 按钮大小: xs ? 'small' : 'middle'
```

**要求：**
- 始终使用 `display: flex` 布局
- 左侧：折叠按钮 + 标题
- 右侧：用户操作按钮
- 小屏幕时按钮使用图标+省略文字的方式

### 2.3 内容区域 (Content)

```typescript
// 响应式边距
margin: xs ? '12px' : sm ? '16px' : '24px 16px'
padding: xs ? 16 : sm ? 20 : 24
```

## 3. 表格响应式规范

### 3.1 列的显示/隐藏策略

```typescript
const columns = [
  { title: 'ID', responsive: ['xs', 'sm', 'md', 'lg', 'xl'] },
  { title: '名称', ellipsis: true }, // 始终显示
  { title: '描述', responsive: ['sm', 'md', 'lg', 'xl'] }, // 小屏隐藏
  { title: '操作', width: xs ? 100 : 250 },
];
```

### 3.2 操作按钮响应式

```typescript
// 小屏幕垂直排列，大屏幕水平排列
<Space size="small" direction={screens.xs ? 'vertical' : 'horizontal'}>
  <Button size={screens.xs ? 'small' : 'middle'}>编辑</Button>
  <Button>{screens.xs ? '' : '删除'}</Button> // 小屏只显示图标
</Space>
```

### 3.3 表格横向滚动

```typescript
<Table
  scroll={{ x: screens.xs ? 600 : undefined }}
  size={screens.xs ? 'small' : 'middle'}
/>
```

## 4. 表单响应式规范

### 4.1 模态框宽度

```typescript
const modalWidth = screens.xs ? '100%' : screens.sm ? 600 : 800;

<Modal
  width={modalWidth}
  style={{ top: screens.xs ? 0 : 100 }}
  bodyStyle={{ padding: screens.xs ? 12 : 24 }}
/>
```

### 4.2 表单项间距

```typescript
// 根据屏幕尺寸调整表单项间距
<Form
  layout="vertical"
  size={screens.xs ? 'small' : 'middle'}
>
```

## 5. 卡片/统计面板响应式

### 5.1 网格布局

```typescript
// 仪表盘的统计卡片
<Row gutter={screens.xs ? [8, 8] : [16, 16]}>
  <Col xs={24} sm={12} md={12} lg={8} xl={6}>
    <Card>...</Card>
  </Col>
</Row>
```

**断点策略：**
- xs: 单列 (24)
- sm/md: 双列 (12)
- lg: 三列 (8)
- xl+: 四列 (6)

## 6. 响应式 Hook 使用规范

### 6.1 基础用法

```typescript
import { Grid } from 'antd';

const { useBreakpoint } = Grid;

const Component: React.FC = () => {
  const screens = useBreakpoint();

  // 判断当前断点
  if (screens.xs) { /* 手机端逻辑 */ }
  if (screens.lg) { /* 桌面端逻辑 */ }

  return (
    <div style={{ fontSize: screens.xs ? 14 : 16 }}>
      内容
    </div>
  );
};
```

### 6.2 条件渲染

```typescript
// 根据屏幕显示/隐藏内容
{!screens.xs && <ExtraInfo />}

// 根据屏幕调整布局
<Space direction={screens.xs ? 'vertical' : 'horizontal'}>
  ...
</Space>
```

## 7. 移动端适配要点

### 7.1 触摸友好

- 按钮最小尺寸：44x44px
- 表格行高适当增加
- 避免过小的点击区域

### 7.2 文字大小

```typescript
// 基础文字大小
fontSize: xs ? 14 : 16

// 标题大小
title: xs ? 18 : 24
```

### 7.3 间距调整

```typescript
// 内边距
padding: xs ? 12 : 24

// 外边距
margin: xs ? 8 : 16

// 组件间距
gutter: xs ? [8, 8] : [16, 16]
```

## 8. 代码示例

### 完整页面响应式模板

```typescript
import React from 'react';
import { Card, Table, Button, Grid, Typography, Space } from 'antd';

const { useBreakpoint } = Grid;
const { Title } = Typography;

const Page: React.FC = () => {
  const screens = useBreakpoint();

  const columns = [
    { title: 'ID', width: screens.xs ? 50 : 60 },
    { title: '名称', ellipsis: true },
    ...(screens.xs ? [] : [{ title: '描述' }]),
    {
      title: '操作',
      render: () => (
        <Space direction={screens.xs ? 'vertical' : 'horizontal'}>
          <Button size={screens.xs ? 'small' : 'middle'}>编辑</Button>
        </Space>
      ),
    },
  ];

  return (
    <Card
      title={<Title level={screens.xs ? 4 : 3}>页面标题</Title>}
      extra={
        <Button size={screens.xs ? 'small' : 'middle'}>
          {screens.xs ? '新建' : '新建条目'}
        </Button>
      }
    >
      <Table
        columns={columns}
        scroll={{ x: screens.xs ? 600 : undefined }}
        size={screens.xs ? 'small' : 'middle'}
      />
    </Card>
  );
};
```

## 9. 检查清单

开发完成后，请检查以下项目：

- [ ] 在 375px (iPhone SE) 宽度下正常显示
- [ ] 在 768px (iPad) 宽度下正常显示
- [ ] 在 1920px (桌面) 宽度下正常显示
- [ ] 侧边栏可以正常折叠/展开
- [ ] 表格可以横向滚动
- [ ] 操作按钮点击区域足够大
- [ ] 文字不会过小或换行异常
- [ ] 模态框在小屏幕下全屏显示
