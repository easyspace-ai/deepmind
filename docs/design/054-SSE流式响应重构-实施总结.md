## 2. 与需求的对应关系

### 2.1 问题 1：流式响应不显示
**解决方式**：
- 使用 aggo 成熟的 `Writer` 实现，替代手动 `fmt.Fprintf`
- `Writer` 内部正确处理 SSE 协议格式和 `flusher.Flush()`
- 使用 `deerflow.Writer` 封装，确保事件正确发送

### 2.2 问题 3：连接不稳定
**解决方式**：
- `Writer` 内置连接状态管理（`IsClosed()` 方法）
- 正确监听 `c.Request.Context().Done()`，客户端断开时及时取消
- 移除复杂的回退逻辑，减少出错可能

---

## 3. 关键实现点

### 3.1 pkg/sse/ - 通用 SSE 包
- **原样复制** aggo 的成熟实现，不做修改
- 包含：
  - `Event` 结构 + 对象池（减少内存分配）
  - `Writer` 结构 + 完整的 SSE 协议实现
  - 多种便捷方法：`WriteEventJSON()`、`WriteData()` 等

### 3.2 pkg/sse/deerflow/ - DeerFlow 扩展层
```go
type Writer struct {
    base *sse.Writer
}

// 三种 DeerFlow 专用方法
func (w *Writer) WriteMessagesTuple(msg interface{}) error
func (w *Writer) WriteValues(values map[string]interface{}) error
func (w *Writer) WriteEnd(usage map[string]interface{}) error
```

### 3.3 langgraph_handler.go 重构
- **添加 import**：`"github.com/weibaohui/nanobot-go/pkg/sse/deerflow"`
- **移除**：手动 `sendEvent` 闭包函数
- **移除**：复杂的回退模拟流式逻辑（`if !useRealStream { ... }`）
- **使用**：`deerflow.NewWriter()` 创建 SSE Writer
- **替换**：所有 `sendEvent(...)` 调用为对应的 `sseWriter.WriteXxx(...)`
