## 1. 设计概述

在现有 Provider 管理页面中增加嵌入模型配置功能，通过弹窗形式管理嵌入模型列表。

## 2. 详细设计

### 2.1 前端 API 扩展

**文件**: `web/src/api/providers.ts`

新增接口：
```typescript
// 获取嵌入模型配置
getEmbeddingModels: (id: number) => Promise<{
  embedding_models: EmbeddingModelInfo[];

updateEmbeddingModels: (id: number, data: {
  embedding_models: EmbeddingModelInfo[];
  default_embedding_model: string;
```

## 3. 交互流程

### 3.1 打开配置弹窗
1. 用户点击"配置嵌入模型"按钮
2. 调用 `providersApi.getEmbeddingModels(provider.id)`
3. 设置 `embeddingModels` 和 `defaultEmbeddingModel` 状态
4. 打开弹窗

### 3.2 添加嵌入模型
1. 用户填写模型ID、名称、维度
2. 点击"添加"按钮
3. 校验数据完整性
4. 添加到 `embeddingModels` 列表
5. 清空输入框

### 3.3 删除嵌入模型
1. 用户点击某行的"删除"按钮
2. 从 `embeddingModels` 列表中移除
3. 如果删除的是默认模型，清空 `defaultEmbeddingModel`

### 3.4 设置默认模型
1. 用户点击某行的"设为默认"按钮
2. 设置 `defaultEmbeddingModel` 为当前模型ID

### 3.5 保存配置
1. 用户点击"保存"按钮
2. 调用 `providersApi.updateEmbeddingModels(provider.id, { embedding_models, default_embedding_model })`
3. 成功提示并关闭弹窗
4. 刷新 Provider 列表

## 5. 错误处理

- API 调用失败显示 `message.error`
- 表单校验失败显示红色提示
- 删除默认模型时提示用户

## 6. 性能考虑

- 嵌入模型列表通常不超过 10 个，无需分页
- 弹窗打开时异步加载数据
- 本地状态管理，保存时才调用 API
