## 1. 设计概述

### 1.1 问题分析

当前对话系统的 context 传递链：
```
Gateway.Start()
  -> context.WithCancel() [gateway.go:198]
    -> Loop.Run(ctx) [run.go:13]
      -> bus.ConsumeInbound(ctx) [run.go:19]
      -> masterAgent.Process(ctx) [run.go:84]
```

问题：所有消息共享同一个 context，无法单独取消某个会话。

### 1.2 解决方案

为每个 session 创建独立的 cancellable context，存储在 session 管理器中，通过 session_key 获取并调用 cancel。

## 2. 详细设计

### 2.1 Session 结构扩展

**文件**: `session/manager.go`

```go
type Session struct {
    Key       string            `json:"key"`
    Messages  []Message         `json:"messages"`
    CreatedAt time.Time         `json:"createdAt"`
    UpdatedAt time.Time         `json:"updatedAt"`

    // 新增字段
    cancel    context.CancelFunc `json:"-"` // context 取消函数，不序列化
    ctx       context.Context    `json:"-"` // 当前会话的 context，不序列化
    mu        sync.RWMutex       `json:"-"` // 保护 cancel 和 ctx 的访问
}
```

新增方法：
```go
// SetContext 设置会话的 context 和 cancel 函数
func (s *Session) SetContext(ctx context.Context, cancel context.CancelFunc)

// GetContext 获取当前会话的 context（如果已取消，返回 nil）
func (s *Session) GetContext() context.Context

// Cancel 取消当前会话
func (s *Session) Cancel() bool

// IsActive 检查会话是否正在活跃处理中
func (s *Session) IsActive() bool
```

### 2.2 Loop 修改

**文件**: `agent/run.go`

修改 `processMessage` 方法：

```go
func (l *Loop) processMessage(parentCtx context.Context, msg *bus.InboundMessage) error {
    // ... 原有代码 ...

    sessionKey := msg.SessionKey()
    sess := l.sessions.GetOrCreate(sessionKey)

    // 为当前会话创建独立的 cancellable context
    ctx, cancel := context.WithCancel(parentCtx)
    sess.SetContext(ctx, cancel)

    // 使用会话独立的 context 处理消息
    response, err := l.masterAgent.Process(ctx, msg)

    // 处理完成后清理 context
    sess.SetContext(nil, nil)

    // ... 原有代码 ...
}
```

### 2.3 SessionManager 新增方法

**文件**: `session/manager.go`

```go
// CancelSession 取消指定会话
func (m *Manager) CancelSession(sessionKey string) bool {
    m.mu.RLock()
    session, ok := m.cache[sessionKey]
    m.mu.RUnlock()

    if !ok {
        return false
    }

    return session.Cancel()
}
```

### 2.4 API 接口

**文件**: `internal/api/server.go` 或新建 `internal/api/session_handler.go`

```go
// SessionHandler 会话处理器
type SessionHandler struct {
    sessionManager *session.Manager
    logger         *zap.Logger
}

// CancelSession 取消会话
func (h *SessionHandler) CancelSession(c *gin.Context) {
    sessionKey := c.Param("session_key")

    success := h.sessionManager.CancelSession(sessionKey)
    if !success {
        c.JSON(http.StatusNotFound, gin.H{
            "success": false,
            "message": "会话不存在或已结束",
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "success": true,
        "message": "会话已取消",
    })
}
```

路由注册：
```go
api := router.Group("/api/v1")
api.POST("/sessions/:session_key/cancel", sessionHandler.CancelSession)
```

## 4. 异常处理

### 4.1 会话不存在
- 返回 404，提示"会话不存在或已结束"

### 4.2 会话已结束
- 如果 session 的 cancel 为 nil，说明会话未在执行中
- 返回 200，但提示"会话不在执行中"

### 4.3 Context 取消后的处理

在 `agent/interruptible.go` 中需要处理 context 取消：

```go
func (i *interruptible) processNormal(ctx context.Context, ...) (string, error) {
    iter := i.adkRunner.Run(ctx, messages, ...)

    for {
        select {
        case <-ctx.Done():
            return "", fmt.Errorf("对话已取消")
        default:
        }

        event, ok := iter.Next()
        // ...
    }
}
```

## 5. 并发安全设计

1. **Session 级别锁**: 每个 Session 有自己的 `sync.RWMutex` 保护 cancel 和 ctx
2. **Manager 读锁**: `CancelSession` 使用读锁获取 session，避免阻塞其他操作
3. **原子操作**: context 的替换和取消操作在锁内完成

## 6. 资源清理

取消后需要确保：
1. ADK Runner 停止执行
2. 会话的 cancel 和 ctx 被清理（设为 nil）
3. 不影响 session 的历史消息和其他状态
