# MCP 渐进式加载设计文档

## 1. 背景与问题

### 1.1 当前现状

目前系统中 MCP 工具的管理方式：
- Agent 绑定 MCP Server 后，该 Server 的所有工具会在对话开始时全部加载
- 工具列表通过 `GetAgentMCPTools` 获取，直接注入到对话上下文中
- 随着 MCP Server 和工具数量增加，上下文会被大量工具定义占用

### 1.2 存在问题

1. **上下文膨胀**：每个 MCP Server 可能包含数十个工具，全部加载会占用大量 token
2. **注意力分散**：大模型需要处理大量工具定义，影响对核心任务的关注
3. **效率降低**：工具调用准确率可能随工具数量增加而下降
4. **缺乏发现机制**：用户/大模型无法按需发现和使用工具

## 2. 设计目标

参考 Skill 的渐进式披露机制，实现 MCP 工具的按需加载：

1. **默认最小化**：对话开始时只展示 MCP Server 名称和简介
2. **按需加载**：通过 `use_mcp` 工具动态加载特定 MCP Server 的工具
3. **会话级缓存**：已加载的 MCP 工具在当前对话中保持可用
4. **向后兼容**：现有 Agent MCP 绑定机制不受影响

## 3. 核心概念

### 3.1 对比：Skill 与 MCP 渐进式加载

| 特性 | Skill 机制 | MCP 渐进式加载 |
|------|-----------|---------------|
| 默认状态 | 展示技能名称列表 | 展示 MCP Server 名称列表 |
| 加载方式 | `use_skill` 工具 | `use_mcp` 工具 |
| 加载参数 | `skill_name` | `server_code` |
| 加载内容 | 技能文档 | MCP 工具列表 + 工具定义 |
| 作用域 | 单次调用，内容注入上下文 | 会话级，工具注册到对话 |

### 3.2 关键组件

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent 对话上下文                          │
├─────────────────────────────────────────────────────────────┤
│  系统提示词                                                  │
│  - 可用 MCP Server 列表（名称+描述）                          │
│  - use_mcp 工具说明                                          │
├─────────────────────────────────────────────────────────────┤
│  动态工具区（use_mcp 调用后扩展）                              │
│  - Server A 的工具 1                                         │
│  - Server A 的工具 2                                         │
│  - Server B 的工具 1                                         │
├─────────────────────────────────────────────────────────────┤
│  基础工具（始终可用）                                         │
│  - read_file, write_file, web_search...                      │
│  - use_skill, use_mcp                                        │
└─────────────────────────────────────────────────────────────┘
```

## 4. 技术设计

### 4.1 数据模型扩展

```go
// AgentMCPBinding 新增字段
type AgentMCPBinding struct {
    // ... 现有字段 ...

    // AutoLoad 是否在对话开始时自动加载该 Server 的所有工具
    // 默认为 false，实现渐进式加载
    AutoLoad bool `json:"auto_load" gorm:"default:false"`

    // LoadedTools 已加载的工具列表（运行时，不持久化）
    LoadedTools []string `json:"-" gorm:"-"`
```

### 4.3 MCP 工具管理器

新增会话级 MCP 工具管理器：

```go
// pkg/agent/tools/mcp/manager.go

type SessionMCPManager struct {
    // 已加载的 MCP Server 及其工具
    loadedServers map[string]*LoadedServer

    // MCP Service 引用
    mcpService mcpsvc.Service

    // 工具注册表（用于动态注册/注销工具）
    toolRegistry *tools.Registry
}

type LoadedServer struct {
    ServerCode string
    ServerName string
    Tools      []tool.InvokableTool
    LoadedAt   time.Time
}

// LoadServer 加载指定 MCP Server 的工具
func (m *SessionMCPManager) LoadServer(serverCode string) (*LoadedServer, error)

// GetLoadedTools 获取所有已加载的工具
func (m *SessionMCPManager) GetLoadedTools() []tool.BaseTool

// IsLoaded 检查 Server 是否已加载
func (m *SessionMCPManager) IsLoaded(serverCode string) bool
```

### 4.4 Agent Loop 集成

修改 `pkg/agent/loop.go`：

```go
type Loop struct {
    // ... 现有字段 ...

    // MCP 会话管理器
    mcpManager *mcp.SessionMCPManager
}

// registerDefaultTools 注册默认工具
func (l *Loop) registerDefaultTools() {
    // ... 现有工具注册 ...

    // 注册 use_mcp 工具
    l.tools.Register(mcp.NewUseMCPTool(l.mcpManager))
}

// GetAvailableToolsForLLM 获取向大模型展示的工具列表
func (l *Loop) GetAvailableToolsForLLM() []tool.BaseTool {
    var result []tool.BaseTool

    // 1. 基础工具（始终包含）
    result = append(result, l.tools.GetTools()...)

    // 2. 自动加载的 MCP 工具
    if l.mcpManager != nil {
        result = append(result, l.mcpManager.GetLoadedTools()...)
    }

    return result
}
```

### 4.5 系统提示词生成

修改系统提示词生成逻辑：

```go
// buildSystemPrompt 构建系统提示词
func (l *Loop) buildSystemPrompt() string {
    var prompt strings.Builder

    // ... 基础提示词 ...

    // MCP Server 列表（仅名称和描述）
    if l.mcpManager != nil {
        servers := l.mcpManager.GetAvailableServers()
        if len(servers) > 0 {
            prompt.WriteString("\n\n## 可用的 MCP Servers\n")
            prompt.WriteString("你可以使用 `use_mcp` 工具加载以下 MCP Server 的工具:\n\n")
            for _, server := range servers {
                prompt.WriteString(fmt.Sprintf("- **%s**: %s\n", server.Code, server.Description))
            }
        }
    }

    // 已加载的 MCP 工具说明
    if l.mcpManager != nil && len(l.mcpManager.GetLoadedServers()) > 0 {
        prompt.WriteString("\n\n## 已加载的 MCP 工具\n")
        for _, server := range l.mcpManager.GetLoadedServers() {
            prompt.WriteString(fmt.Sprintf("\n### %s\n", server.ServerName))
            for _, tool := range server.Tools {
                prompt.WriteString(fmt.Sprintf("- %s\n", tool.Name))
            }
        }
    }

    return prompt.String()
}
```

## 5. 交互流程

### 5.1 正常加载流程

```
用户: 帮我查询今天的天气

大模型: （检查可用 MCP Servers，发现 weather-server）
      我需要使用天气查询工具。

      [调用 use_mcp]
      {
        "server_code": "weather-server",
        "action": "load"
      }

系统:  （加载 weather-server 的工具）
      返回: {
        "server_name": "天气服务",
        "tools": [
          {"name": "get_current_weather", "description": "获取当前天气"},
          {"name": "get_forecast", "description": "获取天气预报"}
        ],
        "message": "MCP Server 'weather-server' 已加载，包含 2 个工具"
      }

大模型: （现在可以使用天气工具了）
      [调用 get_current_weather]
      {"city": "北京"}
```

### 5.2 自动加载配置

对于需要始终可用的 MCP Server，Agent 绑定可以设置 `AutoLoad=true`：

```
Agent 配置:
- MCP Server A: AutoLoad=true （对话开始时自动加载）
- MCP Server B: AutoLoad=false （需要 use_mcp 手动加载）
```

## 8. 实现步骤

1. **数据模型** - 添加 `auto_load` 字段到 `AgentMCPBinding`
2. **后端服务** - 实现 `SessionMCPManager` 和 `use_mcp` 工具
3. **Agent Loop** - 集成 MCP Manager，修改工具列表生成逻辑
4. **前端界面** - 添加 `AutoLoad` 配置开关和加载状态显示
5. **测试验证** - 验证渐进式加载和自动加载两种模式

## 9. 向后兼容性

- 现有 Agent MCP 绑定默认 `auto_load=false`
- 已部署的 Agent 不会自动加载 MCP 工具（需要手动调用 use_mcp）
- API 返回新增字段，不影响旧客户端

## 10. 性能考量

| 场景 | 优化前 | 优化后 |
|------|--------|--------|
| 启动时工具数量 | 50+ (所有 MCP 工具) | 10+ (仅基础工具) |
| 平均上下文长度 | 5000 tokens | 1500 tokens |
| 首次 MCP 调用延迟 | 0ms (已加载) | +50ms (加载耗时) |
| 后续 MCP 调用延迟 | 0ms | 0ms |

## 11. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大模型忘记加载 MCP | 中 | 系统提示词强化说明 |
| 重复加载同一 Server | 低 | Manager 层做幂等处理 |
| 加载失败处理 | 低 | 返回错误信息，大模型可重试 |
| 会话状态丢失 | 低 | 仅影响单次对话，新对话重新加载 |
