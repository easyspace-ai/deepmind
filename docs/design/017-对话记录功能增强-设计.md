## 1. 设计概述

将对话记录页面从单一表格视图升级为多维度数据可视化平台，包括链路可视化、会话聊天视图、统计分析三大核心功能模块。

#### 链路顺序一致性策略

```typescript
const eventPriority: Record<string, number> = {
  llm_call_end: 10,
  tool_completed: 20,
};

const rolePriority: Record<string, number> = {
  tool: 10,
  tool_result: 20,
};

const compareByOrder = (a: ConversationRecord, b: ConversationRecord) => {
  const timeDiff = new Date(a.timestamp).getTime() - new Date(b.timestamp).getTime();
  if (timeDiff !== 0) return timeDiff;
  const eventDiff = (eventPriority[a.event_type || ''] || 1000) - (eventPriority[b.event_type || ''] || 1000);
  if (eventDiff !== 0) return eventDiff;
  const roleDiff = (rolePriority[a.role || ''] || 1000) - (rolePriority[b.role || ''] || 1000);
  if (roleDiff !== 0) return roleDiff;
  return a.id - b.id;
};
```

**设计说明**:
- 主排序遵循时间正序，与“查看对话”一致
- 当时间相同时，通过事件优先级保证 `llm_call_end` 先于 `tool_completed`
- 当角色相同时，通过角色优先级保证 `tool` 先于 `tool_result`
- 通过 `id` 兜底，确保排序稳定

#### 工具结果归并策略

```typescript
// 初始阶段所有节点作为根节点保留，避免节点丢失
// 第二阶段仅处理 tool_result -> tool 的父子归并
sorted.forEach((record, index) => {
  if (record.role !== 'tool_result') return;
  // 优先按 parent_span_id 找最近 tool
  // 其次按 span_id 找最近 tool
  // 最后回退到最近的 tool
  // 匹配成功后将 tool_result 从根级摘除并挂到 tool.children
});
```

**设计说明**:
- 先保留所有节点可见，再做局部父子归并，避免构树错误导致整树被吞
- 归并范围限定在 `tool_result`，减少对 LLM 与普通消息的影响
- 仅允许将 `tool_result` 归并到其之前出现的 `tool` 节点，防止跨时序错挂

**核心逻辑**:
```typescript
// 构建树形结构
const buildTree = (records: ConversationRecord[]): TreeNode[] => {
  const nodeMap = new Map<string, TreeNode>();
  const roots: TreeNode[] = [];

  // 先创建所有节点
  records.forEach(record => {
    nodeMap.set(record.span_id, {
      key: record.span_id,
      title: <TraceNodeTitle record={record} />,
      record,
      children: []
    });
  });

  // 建立父子关系
  records.forEach(record => {
    const node = nodeMap.get(record.span_id);
    if (record.parent_span_id && nodeMap.has(record.parent_span_id)) {
      const parent = nodeMap.get(record.parent_span_id)!;
      parent.children = parent.children || [];
      parent.children.push(node);
    } else {
      roots.push(node);
    }
  });

  return roots;
};
```

```typescript
// 按时间顺序排序，分组为对话轮次
const groupMessages = (records: ConversationRecord[]): ChatRound[] => {
  const sorted = [...records].sort(
    (a, b) => new Date(a.timestamp).getTime() - new Date(b.timestamp).getTime()
  );

  // 简单的分组：user -> assistant/tool 为一轮
  const rounds: ChatRound[] = [];
  let currentRound: ChatRound = { user: null, responses: [] };

  sorted.forEach(record => {
    if (record.role === 'user') {
      if (currentRound.user) {
        rounds.push(currentRound);
      }
      currentRound = { user: record, responses: [] };
    } else {
      currentRound.responses.push(record);
    }
  });

  if (currentRound.user) rounds.push(currentRound);
  return rounds;
};
```

## 5. 数据流设计

### 5.1 链路可视化数据流

```
用户点击"查看链路"
    ↓
前端调用 GET /conversations/trace/:traceID
    ↓
后端返回该 trace 下的所有记录
    ↓
前端按 span_id/parent_span_id 构建树
    ↓
计算相邻节点时间差作为耗时
    ↓
渲染树形组件
```

### 5.2 统计分析数据流

```
用户进入统计页面/点击刷新
    ↓
前端调用 GET /conversations/stats?filters
    ↓
后端聚合查询（按天分组、按 agent 分组等）
    ↓
返回聚合统计数据
    ↓
前端使用 Recharts 渲染图表
```

## 6. 性能考虑

1. **大数据量处理**
   - 链路树：限制单次展示最大 100 个节点
   - 会话对话：虚拟滚动（消息超过 50 条）
   - 统计数据：后端预聚合，避免全表扫描

2. **缓存策略**
   - 统计结果缓存 5 分钟
   - 链路数据缓存 1 分钟

3. **懒加载**
   - 会话列表按需加载（已有分页）
   - 链路详情点击时才加载

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| trace 数据不完整 | 显示警告提示，以扁平列表展示 |
| 统计数据为空 | 显示空状态插图 |
| API 超时 | 显示重试按钮 |
| 筛选无结果 | 提示"无匹配数据，请调整筛选条件" |
