## 1. 设计概述

### 1.1 设计目标

为 User、Channel、Agent 等核心业务实体引入 Code 字段，作为业务层面的唯一标识符。Code 将替代 ID 用于对外交互、日志记录和跨系统通信。

### 1.2 设计原则

1. **全局唯一**：Code 在全局范围内唯一
2. **不可预测**：Code 生成需具有一定的随机性
3. **可读友好**：避免易混淆字符，便于人工识别
4. **向后兼容**：现有 ID 机制保留，渐进式迁移
5. **高性能**：Code 生成和查询性能不劣于 ID

## 3. Code 生成算法设计

### 3.1 算法选择：基于 NanoID 的定制方案

采用 NanoID 思想，但进行以下定制：

1. **自定义字符集**：去除易混淆字符（0, O, 1, I, l）
2. **添加前缀**：标识实体类型（usr_, chn_, agt_）
3. **长度控制**：总长度 12-16 位（含前缀）

### 3.2 字符集定义

```go
const (
    // 去除 0, O, 1, I, l 避免混淆
    CodeAlphabet = "23456789ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz"

    // 各实体前缀
    UserPrefix    = "usr_"
    ChannelPrefix = "chn_"
    AgentPrefix   = "agt_"
)
```

### 3.3 生成算法实现

```go
package codec

import (
    "crypto/rand"
    "math/big"
    "strings"
)

// CodeGenerator Code 生成器
type CodeGenerator struct {
    alphabet []rune
    length   int
}

// NewCodeGenerator 创建生成器
func NewCodeGenerator(length int) *CodeGenerator {
    return &CodeGenerator{
        alphabet: []rune(CodeAlphabet),
        length:   length,
    }
}

// Generate 生成随机 Code
func (g *CodeGenerator) Generate() (string, error) {
    var sb strings.Builder
    alphabetLen := big.NewInt(int64(len(g.alphabet)))

    for i := 0; i < g.length; i++ {
        idx, err := rand.Int(rand.Reader, alphabetLen)
        if err != nil {
            return "", err
        }
        sb.WriteRune(g.alphabet[idx.Int64()])
    }

    return sb.String(), nil
}

// GenerateWithPrefix 带前缀生成 Code
func (g *CodeGenerator) GenerateWithPrefix(prefix string) (string, error) {
    code, err := g.Generate()
    if err != nil {
        return "", err
    }
    return prefix + code, nil
}
```

### 3.4 防碰撞策略

1. **数据库唯一约束**：利用数据库 UNIQUE 索引防止重复
2. **重试机制**：生成冲突时自动重试（最多 3 次）
3. **预生成检查**：在事务中检查 Code 是否已存在

```go
// GenerateUnique 生成唯一 Code（带重试）
func (g *CodeGenerator) GenerateUnique(
    prefix string,
    existsChecker func(string) (bool, error),
) (string, error) {
    for i := 0; i < 3; i++ {
        code, err := g.GenerateWithPrefix(prefix)
        if err != nil {
            return "", err
        }

        exists, err := existsChecker(code)
        if err != nil {
            return "", err
        }

        if !exists {
            return code, nil
        }
    }

    return "", fmt.Errorf("failed to generate unique code after 3 retries")
}
```

### 4.1 Code 服务接口

```go
package service

// CodeService Code 生成服务
type CodeService interface {
    // GenerateUserCode 生成用户 Code
    GenerateUserCode() (string, error)

    // GenerateChannelCode 生成渠道 Code
    GenerateChannelCode() (string, error)

    // GenerateAgentCode 生成 Agent Code
    GenerateAgentCode() (string, error)

    // ValidateCode 验证 Code 格式是否合法
    ValidateCode(code string, prefix string) bool
}
```

### 4.2 实体服务集成

在创建实体时自动生成 Code：

```go
func (s *userService) CreateUser(req CreateUserRequest) (*models.User, error) {
    // 生成 user_code
    userCode, err := s.codeService.GenerateUserCode()
    if err != nil {
        return nil, fmt.Errorf("failed to generate user code: %w", err)
    }

    user := &models.User{
        UserCode: userCode,
        Username: req.Username,
        // ...
    }

    if err := s.userRepo.Create(user); err != nil {
        return nil, err
    }

    return user, nil
}
```

## 6. 数据迁移方案

### 6.1 现有数据迁移

为已有数据生成 Code：

```sql
-- 为现有用户生成 Code（通过数据库触发器或脚本）
UPDATE users
SET user_code = CONCAT('usr_', SUBSTRING(MD5(RAND()), 1, 10))
WHERE user_code IS NULL;
```

### 6.2 迁移脚本

编写 Go 迁移脚本处理数据：

```go
func (s *MigrationService) MigrateUserCodes() error {
    users, err := s.userRepo.ListAll()
    if err != nil {
        return err
    }

    for _, user := range users {
        if user.UserCode == "" {
            code, err := s.codeService.GenerateUserCode()
            if err != nil {
                return err
            }
            user.UserCode = code
            if err := s.userRepo.Update(&user); err != nil {
                return err
            }
        }
    }

    return nil
}
```

### 7.2 上下文传递

在 context 中存储 Code 便于追踪：

```go
// 新增 context key
type UserCodeKey struct{}
type ChannelCodeKey struct{}
type AgentCodeKey struct{}

// 设置/获取方法
func WithUserCode(ctx context.Context, code string) context.Context {
    return context.WithValue(ctx, UserCodeKey{}, code)
}

func GetUserCode(ctx context.Context) string {
    if code, ok := ctx.Value(UserCodeKey{}).(string); ok {
        return code
    }
    return ""
}
```

## 8. 安全考虑

### 8.1 防护措施

1. **Code 不可遍历**：Code 非连续，无法通过递增猜测
2. **访问控制**：敏感操作仍需权限验证，Code 仅作为标识
3. **日志脱敏**：用户 Code 在日志中视为半敏感信息

### 8.2 风险评估

| 风险点 | 等级 | 缓解措施 |
|--------|------|----------|
| Code 碰撞 | 低 | 数据库唯一约束 + 重试机制 |
| Code 预测 | 低 | 随机生成，熵足够 |
| 信息泄露 | 中 | Code 不包含业务信息 |
