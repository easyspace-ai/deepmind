# 020-MCP-Tools管理-需求

## 1. 背景与问题

### 当前状态
- MCP Server 的工具列表存储在 `mcp_servers.capabilities` 字段中（JSON 格式）
- 前端 MCP 管理页面只显示工具数量，没有展示具体工具列表
- Agent 绑定 MCP 时只能绑定到 MCP Server 级别，无法细化到 Tool 级别

### 问题
1. **工具不可见**：用户无法查看 MCP Server 提供了哪些具体工具
2. **管理粒度粗**：无法单独控制每个工具的启用/禁用
3. **数据查询困难**：工具数据存储在 JSON 字段中，难以统计和查询

## 2. 功能目标

### 2.1 核心目标
1. **工具可见性**：在 MCP 管理页面清晰展示每个 MCP Server 的工具列表
2. **独立存储**：将工具信息存储在独立的 `mcp_tools` 表中
3. **两级绑定**：Agent 可以绑定 MCP Server，也可以细化绑定到具体 Tool

### 2.2 用户场景
1. **查看工具**：管理员查看某个 MCP Server 提供的所有工具及其参数定义
2. **刷新工具**：从 MCP Server 重新获取最新的工具列表
3. **精细绑定**：Agent 绑定 MCP 时，可以选择性启用部分工具

## 3. 功能需求

### 3.1 MCP Tools 管理
| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| REQ-001 | 创建独立的 `mcp_tools` 表存储工具信息 | P0 |
| REQ-002 | 工具信息包括：名称、描述、参数定义（JSON Schema） | P0 |
| REQ-003 | 刷新 MCP 能力时，删除旧工具，重新插入新工具 | P0 |
| REQ-004 | 前端展示 MCP Server 的工具列表 | P0 |
| REQ-005 | 工具列表支持查看参数定义详情 | P1 |

### 3.2 Agent 能力配置
| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| REQ-006 | Agent 新增 `mcp_list` 字段，记录可用的 MCP Server 列表 | P0 |
| REQ-007 | 前端统一展示 Skills/Tools/MCP 三种能力的配置 | P0 |
| REQ-008 | Agent 绑定 MCP 时支持选择具体工具 | P0 |
| REQ-009 | 支持全选/取消全选工具 | P1 |
| REQ-010 | 显示已启用工具数量（如 "3/5 个工具"） | P1 |
| REQ-011 | 支持修改已绑定 MCP 的工具选择 | P0 |

### 3.3 工具调用日志
| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| REQ-012 | 创建 `mcp_tool_logs` 表记录工具调用日志 | P1 |
| REQ-013 | 日志包含：会话ID、工具名、参数、结果、耗时、错误信息 | P1 |
| REQ-014 | 前端可查看 MCP Server 的调用历史 | P2 |
| REQ-015 | 日志默认保留 30 天 | P2 |

## 4. 非功能需求

| 需求ID | 需求描述 |
|--------|----------|
| NFR-001 | 工具列表加载响应时间 < 1s |
| NFR-002 | 支持 MCP Server 工具数量无上限 |
| NFR-003 | 工具刷新操作幂等（可重复执行） |

## 5. 约束与假设

### 5.1 约束
- 复用现有 `agent_mcp_bindings.enabled_tools` 字段
- 保持与现有 API 的兼容性

### 5.2 假设
- 同一 MCP Server 下工具名称唯一
- `enabled_tools` 为空表示启用所有工具

## 6. 验收标准

### 6.1 MCP Tools 管理
1. MCP 管理页面可展开查看工具列表
2. 工具列表显示名称、描述、参数定义
3. 刷新按钮可更新工具列表
4. 工具数据存储在独立的 `mcp_tools` 表中

### 6.2 Agent 能力配置
5. Agent 配置页面展示 Skills/Tools/MCP 三种能力
6. Agent 绑定 MCP 时可选择/取消选择工具
7. 绑定列表显示启用的工具数量
8. `mcp_list` 字段正确记录 Agent 可用的 MCP Server

### 6.3 调用日志
9. 工具调用时自动记录日志到 `mcp_tool_logs` 表
10. 前端可查看 MCP Server 的调用历史
