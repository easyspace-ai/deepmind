# 013-嵌入模型配置合并到 LLMProvider - 实现总结

## 概述

成功将 EmbeddingConfig 合并到 LLMProvider 表中，实现 AI 模型配置的集中管理。嵌入模型配置现在统一存储在数据库中，与对话模型共享 API 配置。

## 实现内容

### 1. 数据模型扩展 ✅

**文件**: `internal/models/llm_provider.go`

新增字段:
```go
// 嵌入模型配置（新增）
EmbeddingModels       string `gorm:"type:text" json:"embedding_models,omitempty"`       // JSON数组，包含模型ID、名称、维度
DefaultEmbeddingModel string `gorm:"type:text" json:"default_embedding_model,omitempty"` // 默认嵌入模型ID
```

新增结构体:
```go
// EmbeddingModelInfo 嵌入模型信息
type EmbeddingModelInfo struct {
    ID         string `json:"id"`         // 模型ID
    Name       string `json:"name"`       // 模型名称
    Dimensions int    `json:"dimensions"` // 向量维度
}

// EmbeddingModelConfig 嵌入模型完整配置
type EmbeddingModelConfig struct {
    APIKey     string
    BaseURL    string
    Model      string
    Dimensions int
}
```

新增方法:
- `GetEmbeddingModels()` - 获取嵌入模型列表
- `SetEmbeddingModels()` - 设置嵌入模型列表
- `GetEmbeddingConfig()` - 获取指定模型完整配置
- `GetDefaultEmbeddingConfig()` - 获取默认模型配置
- `HasEmbeddingModels()` - 检查是否配置了嵌入模型

### 2. EmbeddingConfigService 实现 ✅

**文件**: `internal/service/embedding/config.go`

实现了统一的嵌入配置服务接口:

```go
type Service interface {
    GetConfig(ctx context.Context, modelID string) (*Config, error)
    GetDefaultConfig(ctx context.Context) (*Config, error)
    ListProviders(ctx context.Context) ([]*models.LLMProvider, error)
    GetProviderByModel(ctx context.Context, modelID string) (*models.LLMProvider, error)
}
```

特性:
- 从数据库读取嵌入模型配置
- 自动选择第一个可用嵌入模型作为默认
- 支持多 Provider 按优先级选择

接口:
- `NewService(db)` - 创建服务实例

### 3. 配置简化 ✅

**文件**: `config/schema.go`

简化后的 EmbeddingConfig:
```go
type EmbeddingConfig struct {
    Enabled    bool   `json:"enabled"`    // 是否启用向量化
    Model      string `json:"model"`      // 默认模型（可选，留空则从数据库读取）
    Dimensions int    `json:"dimensions"` // 维度（可选，从数据库读取）
}
```

移除字段:
- `APIKey` - 从 LLMProvider 表读取
- `BaseURL` - 从 LLMProvider 表读取

### 4. 环境变量清理 ✅

**文件**: `.env.example`

移除了 AI Provider 环境变量配置:
```bash
# 已移除:
# - OPENAI_API_KEY
# - ANTHROPIC_API_KEY
```

现在所有 AI Provider 配置都通过数据库管理。

### 5. API 扩展 ✅

**文件**: `internal/api/handler_provider.go`

新增端点:
- `GET /api/v1/providers/:id/embedding` - 获取 Provider 嵌入模型配置
- `PUT /api/v1/providers/:id/embedding` - 更新 Provider 嵌入模型配置

支持操作:
- 配置嵌入模型列表（含模型ID、名称、维度）
- 设置默认嵌入模型
- 查询当前嵌入配置状态

### 6. 数据库迁移脚本 ✅

**文件**: `scripts/migrate_embedding.sql`

迁移步骤:
1. 新增 `embedding_models` 和 `default_embedding_model` 字段
2. 为 OpenAI 添加默认嵌入模型配置:
   - text-embedding-3-small (1536维)
   - text-embedding-3-large (3072维)
3. 为 Ollama 添加示例配置:
   - nomic-embed-text (768维)
   - mxbai-embed-large (1024维)
4. 创建索引优化查询性能

## 使用示例

### 初始化时使用
```go
// 创建嵌入配置服务
embeddingSvc := embedding.NewService(db)

// 获取默认嵌入模型配置
cfg, err := embeddingSvc.GetDefaultConfig(context.Background())
if err != nil {
    logger.Warn("获取嵌入模型配置失败", zap.Error(err))
}
// cfg.APIKey, cfg.BaseURL, cfg.Model, cfg.Dimensions
```

### 更新 Provider 嵌入配置
```bash
# 更新嵌入模型配置
curl -X PUT http://localhost:8080/api/v1/providers/1/embedding \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "embedding_models": [
      {"id": "text-embedding-3-small", "name": "Text Embedding 3 Small", "dimensions": 1536}
    ],
    "default_embedding_model": "text-embedding-3-small"
  }'
```
