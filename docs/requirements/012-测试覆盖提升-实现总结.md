
### 命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 单元测试 | `Test<函数>_<场景>_<预期>` | `TestCodeService_GenerateUserCode_Success` |
| 集成测试 | `Test<模块>_<操作>` | `TestAgentRepository_Create` |
| 基准测试 | `Benchmark<操作>` | `BenchmarkGenerateUserCode` |

### 运行命令

```bash
# 运行所有测试
go test ./...

# 运行特定包测试
go test ./internal/service/...

# 运行带覆盖率
go test -cover ./...

# 生成覆盖率报告
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out -o coverage.html
```

## 注意事项

1. **Mock 隔离**: 单元测试必须隔离外部依赖
2. **测试数据**: 集成测试使用独立数据库，避免污染
3. **并行执行**: 测试应支持并行执行 `-parallel`
4. **CI 集成**: 测试应集成到 CI/CD 流程
