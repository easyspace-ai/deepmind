# 需求 010: 性能优化

## 背景

随着数据量增长，部分功能存在性能瓶颈，需要针对性优化。

## 需求列表

### 1. Code 生成重试机制增强

**现状**
文件: `internal/utils/codeutil/codeutil.go`
- 固定重试 3 次
- 无延迟，立即重试
- 高并发时碰撞概率增加

**当前实现**
```go
func GenerateUniqueCodeWithRetry(
    generateFunc func() (string, error),
    checker func(string) (bool, error),
    maxRetries int,
) (string, error) {
    for i := 0; i < maxRetries; i++ {
        code, err := generateFunc()
        // ...
        if !exists {
            return code, nil
        }
    }
    return "", fmt.Errorf("failed after %d retries", maxRetries)
}
```

**需求**
- 支持指数退避（exponential backoff）
- 可配置的重试次数和延迟
- 高并发优化（考虑使用数据库唯一索引避免碰撞）

**验收标准**
- [ ] 添加指数退避：初始 10ms，每次翻倍，最大 1s
- [ ] 重试次数参数化（默认 5 次）
- [ ] 添加详细的日志记录（每次重试记录警告）
- [ ] 考虑使用数据库唯一索引作为最终保障

---

### 2. 数据库索引优化

**现状**
文件: `internal/models/conversation_record.go`
已添加索引的字段：
- `TraceID` - 普通索引
- `EventType` - 普通索引
- `Timestamp` - 普通索引
- `SessionKey` - 普通索引
- `Role` - 普通索引
- `UserCode`, `AgentCode`, `ChannelCode` - 普通索引
- `ChannelType` - 普通索引

**问题**
- 缺少复合索引优化常见查询场景
- 索引命名不规范

**常见查询场景分析**
1. 按用户+时间范围查询对话记录
2. 按 Agent+时间范围统计
3. 按 Channel+时间范围查询
4. 按 SessionKey+时间排序查询

**需求**
- 为高频查询场景添加复合索引
- 规范索引命名

**验收标准**
- [ ] 添加复合索引: `(user_code, timestamp)`
- [ ] 添加复合索引: `(agent_code, timestamp)`
- [ ] 添加复合索引: `(channel_code, timestamp)`
- [ ] 添加复合索引: `(session_key, timestamp)`
- [ ] 索引命名规范: `idx_表名_字段1_字段2`
- [ ] 使用 EXPLAIN 验证索引使用效果

---

### 3. 查询优化：延迟加载大字段

**现状**
文件: `internal/models/conversation_record.go`
- `Content` 字段存储完整的对话内容
- 列表查询时加载了所有字段

**问题**
- 列表查询不需要完整 Content
- 大内容字段影响查询性能

**需求**
- 列表查询使用字段选择，排除大字段
- 详情查询加载完整内容

**验收标准**
- [ ] 在 Repository 层添加字段选择支持
- [ ] 列表查询只选择必要字段（排除 content）
- [ ] 详情查询加载完整字段
- [ ] 对比查询性能提升

---

### 4. 连接池配置优化

**现状**
文件: `internal/database/db.go`
- 使用 GORM 默认连接池配置
- 未针对 SQLite 优化

**需求**
- 配置合适的连接池参数
- SQLite 特定优化（WAL 模式等）

**验收标准**
- [ ] 配置连接池大小（MaxOpenConns, MaxIdleConns）
- [ ] SQLite 启用 WAL 模式
- [ ] 添加忙等待超时配置
- [ ] 监控连接池使用情况

## 优先级

| 需求 | 优先级 | 原因 |
|------|--------|------|
| 数据库索引优化 | P1 | 显著影响查询性能 |
| 查询字段优化 | P1 | 减少不必要的数据传输 |
| Code 生成重试 | P2 | 可靠性提升 |
| 连接池优化 | P2 | 并发性能提升 |
