## 1. 需求背景

当前 LLM Provider 页面支持配置对话模型，但缺少嵌入式模型（Embedding Model）的配置管理功能。后端 API 已支持嵌入模型配置（`GET/PUT /api/v1/providers/:id/embedding`），需要在前端增加对应的管理界面。

## 2. 需求目标

在 Provider 管理页面增加嵌入式模型配置功能，支持：
1. 查看 Provider 的嵌入模型列表
2. 添加嵌入模型（模型ID、名称、维度）
3. 编辑嵌入模型
4. 删除嵌入模型
5. 设置默认嵌入模型

## 3. 功能需求

### 3.1 嵌入模型列表展示

在 Provider 表格或编辑弹窗中展示嵌入模型信息：
- 显示是否配置了嵌入模型（是/否）
- 显示嵌入模型数量
- 显示默认嵌入模型

### 3.2 嵌入模型配置弹窗

点击"配置嵌入模型"按钮打开弹窗：
- 表格展示嵌入模型列表（ID、名称、维度）
- 添加/编辑表单：
  - 模型ID（必填）：如 `text-embedding-3-small`
  - 模型名称（必填）：如 `Text Embedding 3 Small`
  - 维度（必填）：如 `1536`
- 设置默认模型功能
- 删除模型功能

### 3.3 数据格式

嵌入模型 JSON 格式：
```json
[
  {
    "id": "text-embedding-3-small",
    "name": "Text Embedding 3 Small",
    "dimensions": 1536
  },
  {
    "id": "text-embedding-3-large",
    "name": "Text Embedding 3 Large",
    "dimensions": 3072
  }
]
```

## 4. 接口需求

使用已有后端接口：
- `GET /api/v1/providers/:id/embedding` - 获取嵌入模型配置
- `PUT /api/v1/providers/:id/embedding` - 更新嵌入模型配置

请求体格式：
```json
{
  "embedding_models": [
    {
      "id": "text-embedding-3-small",
      "name": "Text Embedding 3 Small",
      "dimensions": 1536
    }
  ],
  "default_embedding_model": "text-embedding-3-small"
}
```

## 5. UI 需求

### 5.1 Provider 列表页
- 新增列"嵌入模型"，显示已配置的模型数量或"未配置"
- 操作列增加"配置嵌入模型"按钮

### 5.2 嵌入模型配置弹窗
- 标题：`配置嵌入模型 - {provider_name}`
- 表格列：模型ID、名称、维度、操作（设为默认/删除）
- 底部添加表单：输入框（ID、名称、维度）+ 添加按钮
- 关闭/保存按钮

## 6. 验收标准

- [ ] 可以在 Provider 列表看到嵌入模型配置状态
- [ ] 可以打开嵌入模型配置弹窗
- [ ] 可以添加新的嵌入模型
- [ ] 可以删除已有嵌入模型
- [ ] 可以设置默认嵌入模型
- [ ] 修改后刷新页面数据保持同步
