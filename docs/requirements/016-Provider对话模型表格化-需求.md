## 1. 需求背景

当前 Provider 编辑弹窗中的"支持的模型"使用 List 组件展示，交互不够直观。用户希望改为与嵌入模型配置相同的表格交互方式，提升用户体验和一致性。

## 2. 需求目标

将 Provider 编辑弹窗中的对话模型配置改为表格展示，支持：
1. 表格展示模型列表（ID、名称、最大Token数）
2. 添加新模型
3. 删除模型
4. 设置默认模型

## 3. 功能需求

### 3.1 模型列表表格

使用 Table 组件替代 List 组件：
- 列：模型ID、名称、最大Token数、默认标识、操作
- 操作列：设为默认、删除

### 3.2 添加新模型

底部添加表单（与嵌入模型样式一致）：
- 模型ID输入框
- 模型名称输入框
- 最大Token数输入框（数字）
- 添加按钮

### 3.3 设置默认模型

- 支持设置默认模型
- 默认模型在表格中显示标识
- 切换默认模型时自动取消之前的默认

## 4. UI 需求

### 4.1 当前样式
```
支持的模型:
[模型ID] [模型名称] [添加]
- 模型A (id-a) [删除]
- 模型B (id-b) [删除]
```

### 4.2 目标样式
```
支持的模型:
模型ID    | 名称    | 最大Token | 默认 | 操作
----------|---------|-----------|------|------
claude-3  | Claude3 | 200000    | 默认 | 删除
gpt-4     | GPT-4   | 128000    | -    | 设为默认 删除

添加新模型:
[模型ID      ] [模型名称      ] [最大Token   ] [添加]
```

## 5. 验收标准

- [ ] 对话模型使用表格展示
- [ ] 支持添加新模型（含ID、名称、最大Token数）
- [ ] 支持删除模型
- [ ] 支持设置默认模型
- [ ] 默认模型在表格中高亮显示
- [ ] 样式与嵌入模型配置保持一致
