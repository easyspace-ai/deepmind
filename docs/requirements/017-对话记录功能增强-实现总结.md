## 3. 关键设计决策

### 3.1 统计查询优化
- 使用 SQL 聚合函数在数据库层完成统计计算，避免大量数据传输
- 每日趋势查询使用 `date(timestamp)` 分组，支持自动时区处理
- 响应时间计算过滤掉超过 5 分钟的异常值，避免统计偏差

### 3.2 链路树构建
- 基于 `span_id` 和 `parent_span_id` 构建树形结构
- 按时间排序确保正确的父子关系
- 计算相邻节点时间差作为节点耗时

### 3.3 聊天界面设计
- 采用左右布局区分用户和助手消息
- 用户消息右对齐蓝色背景，助手消息左对齐白色背景
- 显示每条消息的 token 消耗和时间戳

## 5. 已知限制与待改进

1. **大数据量处理**: 链路树和会话对话未实现虚拟滚动，大量数据时可能影响性能
2. **实时统计**: 当前统计为实时查询，数据量大时响应较慢，后续可考虑缓存
3. **成本计算**: 未实现按模型单价计算成本功能
4. **图表交互**: 图表仅支持基础展示，暂不支持点击查看详情
