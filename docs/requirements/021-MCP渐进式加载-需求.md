# 021-MCP渐进式加载-需求

## 需求概述

实现 MCP 工具的渐进式加载机制，参考 Skill 的 `use_skill` 模式，通过 `use_mcp` 工具按需加载 MCP Server 的工具，减少对话上下文的工具定义数量，提高大模型工具调用效率。

## 问题背景

当前 Agent 绑定 MCP Server 后，该 Server 的所有工具会在对话开始时全部加载到上下文中。随着 MCP Server 和工具数量增加：
- 上下文 token 占用过多
- 大模型注意力分散
- 工具调用准确率下降

## 需求目标

1. 默认只向大模型展示 MCP Server 名称和简介
2. 提供 `use_mcp` 工具用于按需加载指定 Server 的工具
3. 支持配置 AutoLoad，对关键 Server 保持自动加载
4. 已加载的 MCP 工具在当前会话中保持可用

## 功能需求

### FR1: MCP Server 列表展示

**描述**: 对话开始时，系统提示词中只展示 Agent 绑定的 MCP Server 名称和描述。

**验收标准**:
- [ ] 系统提示词包含 "可用的 MCP Servers" 部分
- [ ] 每个 Server 展示 code 和 description
- [ ] 不包含具体的工具定义

### FR2: use_mcp 工具

**描述**: 提供通用工具供大模型调用，加载指定 MCP Server 的工具。

**验收标准**:
- [ ] 工具名称为 `use_mcp`
- [ ] 参数包含 `server_code`（必填）
- [ ] 参数包含 `action`（可选，默认 load）
- [ ] 返回加载的 Server 名称和工具列表
- [ ] 幂等设计：重复加载同一 Server 不报错

### FR3: 会话级 MCP 工具管理

**描述**: 已加载的 MCP 工具在当前对话会话中保持可用。

**验收标准**:
- [ ] 创建 SessionMCPManager 管理已加载的 Servers
- [ ] 支持 GetLoadedTools() 获取所有已加载的工具
- [ ] 支持 IsLoaded(serverCode) 检查 Server 是否已加载
- [ ] 工具在单次对话中保持注册状态

### FR4: AutoLoad 配置

**描述**: Agent MCP 绑定支持配置是否在对话开始时自动加载。

**验收标准**:
- [ ] AgentMCPBinding 模型添加 `auto_load` 字段（bool，默认 false）
- [ ] 前端绑定配置界面添加 AutoLoad 开关
- [ ] API 支持读写 auto_load 字段
- [ ] AutoLoad=true 的 Server 在对话开始时自动加载其工具

### FR5: 已加载工具展示

**描述**: 系统提示词中展示已加载的 MCP 工具，方便大模型使用。

**验收标准**:
- [ ] 系统提示词包含 "已加载的 MCP 工具" 部分
- [ ] 按 Server 分组展示工具名称
- [ ] 工具定义（参数等）注册到对话工具列表

### FR6: 前端状态显示

**描述**: 对话界面显示当前已加载的 MCP Servers。

**验收标准**:
- [ ] 对话界面展示已加载的 MCP Server 列表
- [ ] 显示每个 Server 的工具数量
- [ ] （可选）支持点击加载未加载的 Server

## 非功能需求

### NFR1: 性能
- 加载 MCP Server 工具耗时 < 100ms
- 不增加对话响应延迟（除首次加载外）

### NFR2: 兼容性
- 向后兼容现有 Agent MCP 绑定
- 默认 auto_load=false，不影响现有行为

### NFR3: 可靠性
- MCP Server 加载失败返回友好错误信息
- 支持重试加载

## 接口变更

### API 变更

```
GET /api/v1/agents/:id/mcp-bindings
响应增加: auto_load (bool)

PUT /api/v1/agents/:id/mcp-bindings/:binding_id
请求支持: auto_load (bool)
```
