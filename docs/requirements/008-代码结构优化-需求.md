# 需求 008: 代码结构优化

## 背景

近期代码变更较多，部分代码存在重复逻辑和效率问题，需要进行结构优化。

## 需求列表

### 1. 分页逻辑提取为通用函数

**现状**
- 文件: `internal/service/conversation/service.go`
- 多个方法包含相同的分页参数校验代码:
  ```go
  if page < 1 {
      page = 1
  }
  if pageSize < 1 {
      pageSize = 20
  }
  if pageSize > 1000 {
      pageSize = 1000
  }
  ```

**影响范围**
- `internal/service/conversation/service.go`: `ListBySessionKey`, `ListByTimeRange`, `ListRecent`
- 其他服务中的类似代码

**需求**
- 创建通用的分页参数处理函数
- 支持自定义默认值和最大值
- 统一的错误返回

**验收标准**
- [ ] 创建 `internal/utils/pagination` 包
- [ ] 实现 `NormalizePageParams(page, pageSize, defaultSize, maxSize) (int, int)`
- [ ] 替换 conversation 服务中的重复代码
- [ ] 检查并替换其他服务中的重复代码

---

### 2. 数据迁移脚本 Provider 映射修复

**现状**
- 文件: `scripts/migrate_data.go:247-248`
- Provider 的 UserCode 映射使用假设关系:
  ```go
  p.UserCode = userCodeMap[p.ID] // 假设 Provider ID 对应 User ID
  ```

**问题**
- Provider ID 不一定等于 User ID
- 可能导致错误的用户关联

**需求**
- 从旧数据库中查询 Provider 与 User 的真实关联关系
- 正确映射 UserCode

**验收标准**
- [ ] 修复 Provider 迁移逻辑
- [ ] 从旧数据库读取 Provider 的 UserID 字段
- [ ] 验证迁移后的数据正确性

---

### 3. Repository 层批量查询优化（N+1 查询）

**现状**
- 文件: `internal/api/handler_conversation.go:48-58`
- 循环查询数据库获取 Agent 和 Channel 名称:
  ```go
  for code := range agentCodes {
      if agent, err := h.agentService.GetAgentByCode(code); err == nil && agent != nil {
          agentNames[code] = agent.Name
      }
  }
  ```

**需求**
- 在 Repository 层支持批量查询（`GetByCodes`）
- 使用 IN 查询替代循环单次查询
- 保持现有 API 兼容性

**验收标准**
- [ ] 在 `repository/agent.go` 添加 `GetByCodes(codes []string) ([]*Agent, error)`
- [ ] 在 `repository/channel.go` 添加 `GetByCodes(codes []string) ([]*Channel, error)`
- [ ] 修改 `enrichConversationRecords` 使用批量查询
- [ ] 性能测试对比（N+1 vs 批量查询）

---

### 4. Code 生成重试机制优化

**现状**
- 文件: `internal/utils/codeutil/codeutil.go`
- 固定重试 3 次，无退避策略
- 高并发或数据量大时可能失败

**需求**
- 支持可配置的重试次数
- 增加指数退避延迟
- 优化碰撞概率（可考虑增加 Code 长度）

**验收标准**
- [ ] 重试次数可配置（参数化）
- [ ] 添加指数退避延迟（backoff）
- [ ] 添加详细的错误日志

## 优先级

| 需求 | 优先级 | 原因 |
|------|--------|------|
| N+1 查询优化 | P1 | 性能影响明显 |
| 分页逻辑提取 | P2 | 代码质量提升 |
| Provider 映射修复 | P2 | 数据正确性 |
| Code 生成优化 | P2 | 可靠性提升 |
