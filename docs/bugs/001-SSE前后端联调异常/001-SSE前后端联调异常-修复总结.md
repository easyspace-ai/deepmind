# SSE前后端联调异常 - 修复总结

## 1. 修复目标

- 稳定后端 SSE 输出
- 让 `frontend/` 与 `/api/langgraph` 的联调链路回到一致协议
- 暂时移除未落地的恢复能力依赖

## 2. 计划修复项

- 修正 `pkg/sse/deerflow/writer.go` 事件名和事件体
- 重构 `internal/api/langgraph_handler.go` 的写流与结束路径
- 调整 `frontend/src/core/threads/hooks.ts` 的流式提交选项

## 3. 验证方式

- Go 侧构建和定向测试
- 前端 TypeScript 检查
- 人工联调聊天页与 Agent 聊天页

## 4. 实际完成项

- 已将 DeerFlow SSE 主事件恢复为标准事件名：
  - `messages-tuple`
  - `values`
  - `end`
- 已保留 `metadata` 事件，方便客户端调试与扩展
- 已为 `pkg/sse.Writer` 增加串行写保护，减少并发写流风险
- 已统一 `streamRun` 的结束路径，避免重复结束和不完整响应
- 已让 `pkg/deerflow/client` 同时兼容旧事件名与新事件名
- 已在 `frontend/` 中关闭 `streamResumable`，避免 `joinStream` 未实现时的能力错配
- 已补充设计文档与缺陷文档

## 6. 已知限制

- 仓库根 `make setup` 当前会扫描非 Go 目录并触发无效 import path
- 仓库根 `make build` 仍引用不存在的 `cmd/nanobot`
- `joinStream` 尚未实现，因此本次以前端关闭恢复能力作为过渡方案

## 7. 安全自检

- 本次未新增外部输入入口
- 未扩大权限范围，也未引入新的高权限默认配置
- 主要安全收益在于降低流式响应异常中断导致的不一致状态风险
