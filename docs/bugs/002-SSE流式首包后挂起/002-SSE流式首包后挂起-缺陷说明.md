# SSE 流式首包后挂起 - 缺陷说明

| 项目 | 内容 |
|------|------|
| 缺陷编号 | 002 |
| 创建日期 | 2026-03-25 |
| 状态 | 修复中 |

## 1. 缺陷现象

- 前端发送聊天请求后，只能收到 `metadata` 首包，随后页面提示 `network error`
- 后端日志出现上游模型请求 `context canceled`
- 使用 `curl -N` 直连 `/api/langgraph/threads/:threadId/runs/stream` 时，也只能看到首个 `metadata` 事件，后续长期无 `messages-tuple`、`values`、`end`

## 2. 影响范围

- `frontend/` 与 `/api/langgraph` 的联调
- DeerFlow/LangGraph 兼容流式协议
- 纯后端 SSE 流接口可用性

## 3. 复现方式

```bash
curl -N --max-time 25 -X POST http://127.0.0.1:7101/api/langgraph/threads/<thread_id>/runs/stream \
  -H 'Content-Type: application/json' \
  -H 'Accept: text/event-stream' \
  -d '{"assistant_id":"lead_agent","messages":[{"type":"human","content":[{"type":"text","text":"调研一下deer-flow"}]}]}'
```

预期：持续收到 `messages-tuple`、`values`、`end`

实际：仅收到 `metadata`，随后连接长时间静默并最终超时
